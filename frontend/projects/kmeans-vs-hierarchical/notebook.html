Comparison 1 - Cluster Shape and Outliers¶

We see both models agree on the general structure of the clusters. Some ambiguous points may be assigned differently, but the overall clusters are similar. This is expected since Ward linkage minimizes variance and K-means minimizes squared distances - both favor compact clusters. K-means is more sensitive to outliers because they pull centroids away from the true cluster center.

We see k-means algorithm produces different results each run due to random initialization, while hierarchical algorithm is deterministic. K-means can converge to local minima depending on starting centroids, which is why sklearn runs it multiple (10) times by default.

Comparison 3 - Computational Complexity¶

Hierarchical clustering has complexity O(N^3), or O(N^2) with optimization. K-means has complexity O(KNI) — linear in N per iteration.

With N = 1,000,000 data points, K-means would require roughly 3 × 20 × 1,000,000 = 60 million operations. Hierarchical would require 1,000,000^2 = 1 trillion operations minimum.

Takeaway: For smaller datasets, hierarchical clustering is a good choice as it's deterministic and robust. For large datasets, K-means is preferable since it scales linearly with N, while hierarchical scales quadratically (or worse).

Conclusion¶

Both algorithms found similar clusters in the wine dataset, which suggests the data has well-defined groupings. K-means is faster and scales better to large datasets, but requires careful initialization and is sensitive to outliers.

Hierarchical clustering is deterministic and lets you explore different numbers of clusters through the dendrogram, but becomes impractical for large datasets due to its quadratic complexity.

For this dataset (178 samples), either method works well. For millions of data points, K-means would be the clear choice.
